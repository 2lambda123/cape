1""" 

2:mod:`cape.pyus.runmatrix`: US3D run matrix module  

3================================================== 

4 

5This module handles classes and methods that are specific to the list of run 

6cases (i.e., the "trajectory"). It is defined in the ``"RunMatrix"`` section 

7of the master JSON file (e.g. :file:`pyUS.json`), and is usually read from a 

8modified comma-separated or space-separated text file. 

9 

10The primary FUN3D state variables are Mach number, angle of attack, and 

11sidelsip. To this it is common practice to add a ``"config"`` variable (which 

12sets the name of the group folder) and a ``"Label"`` which can be used to give 

13an extension to the name of a case. A run matrix using only these variables 

14could be defined as follows. 

15 

16 .. code-block:: javascript 

17  

18 "RunMatrix": { 

19 "Keys": ["mach", "alpha", "beta", "q", "T", "config", "Label"], 

20 "File": "inputs/matrix.csv" 

21 } 

22  

23Then the matrix file :file:`inputs/matrix.csv` could have contents such as the 

24following. 

25 

26 .. code-block:: none 

27  

28 # mach, alpha, beta, q, T, config, Label  

29 0.8, 0.0, 0.0, 480.00, 425.7, ascent, 

30 1.2, 2.0, 0.0, 550.00, 425.7, ascent, a 

31 

32Then the following example API commands could be used to show the case names. 

33In this case *fun3d* is the interface to a FUN3D run configuration read from 

34a JSON file, and *fun3d.x* is the run matrix. 

35 

36 .. code-block:: pycon 

37  

38 >>> import cape.pyus 

39 >>> cntl = pyUS.Cntl("pyUS.json") 

40 >>> cntl.x.GetFullFolderNames() 

41 ['ascent/m0.8a0.0b0.0', 'ascent/m1.2a2.0b0.0_maxq'] 

42  

43For this module, there are no methods that are particular to FUN3D. All 

44functionality is inherited from :class:`cape.runmatrix.RunMatrix`. 

45 

46:See Also: 

47 * :mod:`cape.runmatrix` 

48 * :mod:`cape.pyus.fun3d` 

49""" 

50
